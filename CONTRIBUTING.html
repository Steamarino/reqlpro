    • ReQLPro Contributor Guide

    Thanks for checking out our contributor guide! You can help us out in a few different ways:

    • Open an issue to let us know about errors you're facing or to make suggestions for future development
    • Fork this repository and submit a pull request
    • Improve the documentation (coming soon, see Resources below for now)

    To submit a pull request, fork the ReQLPro repository and then clone your fork:

    git clone git@github.com:<your-name>/reqlpro.git

    Make your suggested changes, git push and then submit a pull request. Note that before we can accept your pull requests, you need to sign our Contributor License Agreement.

    Please consult our Development Guide for more information on working on ReQLPro locally.
